§ 01 — Notes on the practice
On design systems, in a few sentences.

A short set of positions I keep returning to as I build. Longer versions live inside the company; the shape of them is here.

i.

A design system is not a component library.

It is a shared language for how the product feels. Components are the sentences; tokens, motion, tone, and rhythm are the grammar. If the grammar is right, the sentences write themselves.

ii.

The best tokens are the ones designers stop noticing.

A token that fits is invisible — reached for like a keyboard shortcut, not consulted like a specification. If people still Cmd-F for hex values, the system has more work to do.

iii.

Themes are not skins.

Themes are the system's admission that different products, moments, and audiences need different rooms — same house, different rooms. Treat a theme as a first-class object, versioned and shippable, not a set of overrides.

iv.

The Figma–code seam is where the system lives or dies.

Design that can't leave Figma is a mood board. Code that can't be redrawn in Figma is a black box. The tooling that keeps the two mutually legible is not overhead — it is the system.
